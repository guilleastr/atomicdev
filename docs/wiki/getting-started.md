# Atomic Dev Overview

This is the Atomic Dev framework. It's a skill-based agent system for your development workflow.

> **New to this?** Start with [Framework Skills](./guides/framework-skills.md) to understand how the system works, then check [Configuration & State](./guides/configuration.md) for setup details.

## How It Works

**You describe what you want. Agents plan, implement, and document.**

```
You:        "Add user authentication"
             ↓
Planner:    Maps context, writes plan, 3 judges assess confidence
             ↓
Coder:      Implements, verifies, explains
             ↓  
You:        Accept or reject
             ↓
Commit:     Docs Creator updates documentation, writes commit message
             ↓
Result:     Code + documentation committed together, automatically
```

The five baseline skills orchestrate this workflow. See [Framework Skills](./framework-skills.md) for details.

## Key Concepts

### Skills
Self-contained units with a specific responsibility. Five baseline skills (Core, Explorer, Planner, Implementer, Docs Creator) plus domain-specific skills as your repo grows.

### Work Units
One unit of work. Starts when you ask the agent to do something, ends when you accept or reject.  Decisions are preserved in documentation.

### Deliberation
When agents disagree or confidence is low, three judges (Pragmatist, Architect, Skeptic) discuss the plan. Dissent is recorded—never discarded.

## How Documentation is Maintained

Documentation is automatically generated by **Docs Creator** when you:
- Commit code (via git hook)
- Say "update wiki" or "update docs"

The wiki mirrors your repository structure with readable file names and semantic folders. It grows with your project—clean, lean, and always current.
